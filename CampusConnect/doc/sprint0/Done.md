What counts as done for the project:

        A user story is considered done when- 

* All features are implemented according to the user’s request.  
* The code written is functional and edge cases are tested by all members of the team.  
* Code has been committed to the main branch.  
* Code has compiled without any errors.  
* The user story has documentation written for it.  
* At the end of each sprint, the feature is ready.  
* The feature is integrated with the front-end and back-end systems.
